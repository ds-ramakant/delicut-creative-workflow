================================================================================
CREATIVE WORKFLOW — MICRO-STEP, HUMAN-IN-THE-LOOP
================================================================================

PHILOSOPHY
----------
- Micro-steps over batch workflows
- Quality over speed
- Human approval gate at every stage before advancing
- Image and copy are fully decoupled — clean canvas images, separate copy file
- Each iteration produces exactly 4 variants
- Engine choice is made by the user per variant before any API call is made


SESSION ID FORMAT
-----------------
Format: {persona_shortname}_{MMDD-HHMM}
Example: harry_0317-1242

Persona shortnames: harry / mansoor / nikky


--------------------------------------------------------------------------------
STAGE 1 — CREATIVE DIRECTION
--------------------------------------------------------------------------------
Who: Claude (me), based on brief row + creative_dna.json + competitor_audit.json
What: Synthesise a visual concept, mood, composition style, color approach,
      do's and don'ts for the selected brief row
Output: Written creative direction presented to user for approval

Gate: User approves or redirects the direction before Stage 2 begins


--------------------------------------------------------------------------------
STAGE 2 — 4 PROMPT VARIANTS
--------------------------------------------------------------------------------
Who: Claude (me)
What: Write 4 distinct image prompts, each taking a different creative angle
      on the approved direction.
      Every prompt explicitly instructs the engine to produce:
      - NO text
      - NO copy overlays
      - NO headlines or captions baked into the image
      Clean canvas only — all copy lives in adcopy_log.xlsx
Output: 4 prompts presented to user

Gate: User approves, kills, or tweaks each prompt before Stage 3


--------------------------------------------------------------------------------
STAGE 3 — ENGINE SELECTION
--------------------------------------------------------------------------------
Who: User
What: Script writes 4 rows to session_log.xlsx with prompts pre-filled.
      Engine column is left blank.
      User opens session_log.xlsx, assigns an engine to each variant (A/B/C/D),
      saves the file, and confirms to proceed.
      Available engines: Midjourney / Gemini / Ideogram / Magic Media
      Rule: One brief row uses one engine per variant — user chooses freely.
Output: session_log.xlsx with engine column filled by user

Gate: User confirms engine choices before any API call is made


--------------------------------------------------------------------------------
STAGE 4 — GENERATE + REVIEW
--------------------------------------------------------------------------------
Who: Script (generation) + User (review)
What: Images are generated per variant using the chosen engine.
      Only session_log.xlsx is written at this stage.
      adcopy_log.xlsx is NOT created here — copy is generated separately
      after approval so that it is inspired by the actual approved image.

  FILE — outputs/session_log.xlsx (image tracking + feedback)
  Columns:
    session_id   | persona | iteration | variant | engine | dimensions |
    prompt       | image_file | score   | notes   | status

  - score:  user fills 1–5 after reviewing the image
  - notes:  user fills free-text feedback comments
  - status: user sets to pending / needs-refinement / approved

Output: 4 images saved locally, session_log.xlsx updated

Gate: User opens session_log.xlsx, scores each image (1–5), adds notes,
      sets status, saves file, and tells Claude "done"


--------------------------------------------------------------------------------
STAGE 5 — ITERATE OR APPROVE
--------------------------------------------------------------------------------
Who: Claude (prompt refinement) + Script (regeneration)
What: For each variant:
      - Score < 4 → Claude refines the prompt based on user notes →
        regenerate that variant only → new row appended to session_log.xlsx
      - Score >= 4 + status = approved → image advances to Stage 6
      Full history preserved in session_log.xlsx across all iterations.
Output: Refined images + updated rows in session_log.xlsx

Gate: Explicit per-image approval (score >= 4 + status = approved)
      before any image moves to Stage 6


--------------------------------------------------------------------------------
STAGE 6 — AD COPY GENERATION
--------------------------------------------------------------------------------
Who: Script (step7_generate_adcopy.py) — Claude Vision + Claude copy
What: Triggered only after image approval. For each approved image:
      1. Claude Vision reads the approved image
      2. Identifies the dominant emotion, subject, and visual moment
      3. Generates image_text — punchy overlay copy inspired directly by
         what is visible in the image (not the brief hook)
      4. Generates ad_descriptors — structured copy package:
           Headline: ≤30 characters
           Subline:  ≤90 characters
           CTA:      short action phrase
           Offer:    "25% Off your first plan | Use code: BETTERYOU"
      5. Both outputs are informed by copy_dna.json — patterns extracted
         from Delicut's top-performing ads via step3b

  FILE — outputs/adcopy_log.xlsx (copy for designer handoff)
  Columns:
    session_id   | persona | iteration | variant | engine | dimensions |
    image_file   | image_text | ad_descriptors

  - image_text:     bold overlay copy placed directly ON the image
  - ad_descriptors: headline / subline / CTA / offer — outside the frame,
                    added manually by designers in Figma

Output: adcopy_log.xlsx populated with one row per approved image

Gate: User reviews copy before Figma handoff


--------------------------------------------------------------------------------
STAGE 7 — FIGMA HANDOFF
--------------------------------------------------------------------------------
Who: Script (Figma API)
What: For each approved image:
      - Clean image pushed into Figma master template as a background layer
      - image_text from adcopy_log.xlsx placed as overlay text layer
      - ad_descriptors (headline/subline/CTA/offer) available as reference
        — designers apply and adjust these freely outside the frame
      - Frame exported in all 6 size formats:
        1080x1080, 1080x1920, 1920x1080, 1200x628, 1080x1350, 960x1200
Output: Figma frames ready for designer review

Gate: User reviews the Figma frame before export is triggered
